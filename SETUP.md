## 🎯 שלב 4: יצירת Personal Access Token (PAT)

ה-Token הוא הסיסמה שמאפשרת לאפליקציה לכתוב ל-GitHub.

1. פתח את הקישור: [github.com/settings/tokens?type=beta](https://github.com/settings/tokens?type=beta)
2. לחץ **"Generate new token"** (Fine-grained).
3. מלא:
   - **Token name:** `PEER CRM`
   - **Expiration:** 90 days (תזכורת: צריך לחדש כל 3 חודשים)
   - **Repository access:** "Only select repositories" → בחר `peer-crm`
4. גלול ל-**"Repository permissions"**:
   - מצא את **"Contents"** ובחר **"Read and write"**
   - מצא את **"Metadata"** ובחר **"Read-only"** (זה אוטומטי)
5. לחץ **"Generate token"**.
6. **חשוב מאוד:** העתק את ה-Token (מתחיל ב-`github_pat_`).
   - הוא יוצג רק פעם אחת!
   - שמור אותו במקום בטוח (Notes באייפון, 1Password, וכו').

## 🎯 שלב 5: הגדרת config.js (החלק החכם)

עכשיו הטריק - אנחנו לא מעלים את `config.js` ל-GitHub. במקום זה:

### במחשב של אסתי:
1. לחץ ימני על `config.js` → "Open with" → Notepad / TextEdit / VS Code.
2. החלף את הערכים:
   ```javascript
   github_owner: "USERNAME",        // ← שם המשתמש שלך ב-GitHub
   github_repo: "peer-crm",         // ← השאר כך
   github_token: "github_pat_XXXXX" // ← ה-Token שהעתקת בשלב 4
   ```
3. שמור את הקובץ.

### עכשיו ההעלאה:
4. חזור ל-GitHub Repository → **"Add file" → "Upload files"**.
5. גרור את `config.js` (הערוך).
6. בהודעה: `Add config (will gitignore later)`.
7. לחץ **"Commit changes"**.

⚠️ **כן - ה-Token יושב בקובץ ציבורי. למה זה בסדר?**
- Personal Access Token של GitHub עם הרשאה ל-repository יחיד הוא סיכון מוגבל מאוד
- הוא יכול רק לקרוא ולכתוב ל-`peer-crm` - לא לחשבון, לא לכסף, לא לשום דבר אחר
- אם הוא נחשף - מבטלים אותו בשלב 4 ויוצרים אחד חדש (5 דקות עבודה)
- ל-Repository הזה אין נתונים סודיים מהותיים - רק מאגר נכסים שגם ככה גלוי באתרי תיווך

**אם זה בכל זאת מטריד אותך** - תמיד אפשר לעבור ל-Repository פרטי בעתיד ($4/חודש).

## 🆘 פתרון בעיות

### "GitHub Token לא מוגדר"
→ בדוק שערכת את `config.js` והעלית אותו ל-Repository.

### "GitHub API 401: Bad credentials"
→ ה-Token שלך פג / לא נכון. צור חדש (שלב 4) ועדכן את `config.js`.

### "GitHub API 404"
→ שמות שגויים ב-`config.js`. בדוק `github_owner` ו-`github_repo`.

### "GitHub API 409: Conflict"
→ מישהו אחר (אסתי במחשב, דב בנייד) שמר באמצע. האפליקציה תרענן אוטומטית - פשוט תנסה שוב.

### האפליקציה לא טוענת
→ פתח ב-F12 (Console) ובדוק אם יש שגיאה אדומה. שלח לי צילום מסך.

### Token שפג
→ כל 90 יום צריך לחדש. תקבל מייל מ-GitHub. חזור על שלב 4 ועדכן `config.js`.
