## 🔐 איך הבטיחות עובדת (חשוב להבין!)

ה-Token של GitHub הוא **כמו סיסמה** - אסור שיהיה בקוד הציבורי.

**הפתרון שלנו:** בפעם הראשונה שאסתי פותחת את האפליקציה במכשיר, היא תתבקש להדביק את ה-Token פעם אחת. הוא יישמר ב-`localStorage` של הדפדפן באותו מכשיר בלבד - **לא בקובץ, לא ב-GitHub, לא בענן**.

✅ אם בעתיד מישהו זר ימצא את ה-Repository הציבורי - אין שם Token, אין סיכון.
✅ אסתי תזין את ה-Token פעם אחת בנייד, פעם אחת במחשב - וזהו, גמרנו.
✅ אם ה-Token פג / נחשף - מבטלים אותו ב-GitHub וזהו.

## 🎯 שלב 1: יצירת Repository ב-GitHub

1. היכנס ל-[github.com](https://github.com) והתחבר.
2. לחץ על הכפתור הירוק **"New"** (או [github.com/new](https://github.com/new)).
3. מלא:
   - **Repository name:** `peer-crm`
   - **Description:** `PEER Luxury Real Estate CRM`
   - **Public** ✅ (חינמי - אין סודות בתוכו!)
   - ✅ **Add a README file**
4. לחץ **"Create repository"**.

## 🎯 שלב 2: עריכת config.js לפני העלאה

לפני שמעלים, צריך לערוך את `config.js` עם שם המשתמש שלך:

1. פתח את `config.js` ב-Notepad / TextEdit / VS Code.
2. החלף את `YOUR_GITHUB_USERNAME` בשם המשתמש שלך ב-GitHub.

   ```javascript
   github_owner: "esti-heller",   // ← שים כאן את שם המשתמש שלך
   github_repo: "peer-crm",       // ← השאר כך (אם זה השם שבחרת)
   ```

3. שמור את הקובץ.

⚠️ **שים לב:** אין כאן Token! זה בכוונה. הקובץ הזה בטוח לפרסום.

## 🎯 שלב 3: העלאת הקבצים

1. ב-Repository החדש, לחץ **"Add file" → "Upload files"**.
2. גרור את כל הקבצים האלה:

## 🎯 שלב 4: הפעלת GitHub Pages

1. ב-Repository, לחץ על **"Settings"** (חלק עליון).
2. בתפריט הצד השמאלי, לחץ **"Pages"**.
3. תחת **"Source"**, בחר **"Deploy from a branch"**.
4. **Branch:** `main`, **Folder:** `/ (root)`.
5. לחץ **"Save"**.
6. חכה 1-2 דקות. תקבל כתובת ירוקה כמו:
   `https://USERNAME.github.io/peer-crm/`

⚠️ **שמור את הכתובת הזו - זו הכתובת של האפליקציה.**

## 🎯 שלב 5: יצירת Personal Access Token (PAT)

1. פתח את הקישור: [github.com/settings/tokens?type=beta](https://github.com/settings/tokens?type=beta)
2. לחץ **"Generate new token"** (Fine-grained).
3. מלא:
   - **Token name:** `PEER CRM`
   - **Expiration:** 90 days (תזכורת: צריך לחדש כל 3 חודשים)
   - **Repository access:** "Only select repositories" → בחר `peer-crm`
4. גלול ל-**"Repository permissions"**:
   - מצא את **"Contents"** ובחר **"Read and write"**
   - **"Metadata"** ייבחר אוטומטית כ-Read-only
5. לחץ **"Generate token"**.
6. **חשוב מאוד:** העתק את ה-Token (מתחיל ב-`github_pat_`).
   - הוא יוצג רק פעם אחת!
   - שמור אותו זמנית במקום נגיש (Notes / WhatsApp לעצמך)

## 🆘 פתרון בעיות

### "GitHub Token לא מוגדר" / החלון של ה-Token ממשיך להופיע
→ לחץ Cancel, ופתח את הדפדפן מחדש. אם זה ממשיך - בדוק שב-`config.js` עדכנת את `github_owner` נכון.

### "GitHub API 401: Bad credentials"
→ ה-Token שלך פג / לא נכון. צור חדש (שלב 5) ובמערכת לחץ "החלף Token".

### "GitHub API 404"
→ שמות שגויים ב-`config.js`. בדוק `github_owner` ו-`github_repo`.

### "GitHub API 409: Conflict"
→ מישהו אחר (אסתי במחשב, דב בנייד) שמר באמצע. האפליקציה תרענן אוטומטית - פשוט תנסה שוב.

### האפליקציה לא טוענת
→ פתח ב-F12 (Console) ובדוק אם יש שגיאה אדומה.

### Token שפג
→ כל 90 יום צריך לחדש. תקבל מייל מ-GitHub. צור Token חדש (שלב 5) → באפליקציה לחץ "החלף Token".

### אסתי מנקה היסטוריית דפדפן
→ ה-Token יימחק מהמכשיר. תצטרך להזין שוב פעם אחת. (זה הסיבה שעדיף לשמור את ה-Token גם במייל / Notes).

## 🛡 אבטחה - מה אם ה-Token נחשף?

אם בטעות ה-Token פורסם איפשהו (פוסט בפייסבוק, סקרין שייר, וכו'):

1. היכנס ל-[github.com/settings/tokens](https://github.com/settings/tokens)
2. מצא את ה-Token "PEER CRM" → לחץ **"Delete"**.
3. צור Token חדש (שלב 5).
4. באפליקציה: לחץ "החלף Token" והכניס את החדש.
